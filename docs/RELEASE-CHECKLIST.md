# Release Checklist

Use this checklist before publishing a SignalTrim release.

## Preflight

- Confirm `package.json` version matches the intended tag.
- Confirm README badges match package metadata.
- Confirm `CHANGELOG` or release notes are prepared.
- Run local verification:

```bash
npm test
npm run smoke
npm run check:install-surfaces
npm run test:python
npm run verify
npm run pack:dry
```

## Package Checks

- `npm pack --dry-run --json` includes `bin/`, `src/`, `skills/`, `plugins/`, `commands/`, `docs/`, `README.md`, and `LICENSE`.
- Package does not include caches, `__pycache__`, or local temp files.
- `bin.signaltrim` and `bin.signaltrim-shrink` targets exist.
- Node engine remains `>=20.19`.

## Provenance

- Release commit is on `main`.
- CI is passing for the release commit.
- `dist/release/checksums.sha256` is generated by the release workflow.
- GitHub release artifacts include npm tarball and source archives.
- GitHub release artifacts include platform launch bundles for Windows, macOS, and Linux.
- Platform bundles are Node launch packs, not native compiled executables; Node 20.19+ remains required.

## Publish

```bash
git tag v0.6.4
git push origin v0.6.4
```

If publishing to npm later:

```bash
npm publish --provenance
```

## Rollback

- Unpublish only if the package is harmful and npm policy allows it.
- Otherwise publish a patch release.
- Mark the bad GitHub release as prerelease or add a warning note.
- Open a public issue with the exact broken install path and workaround.
